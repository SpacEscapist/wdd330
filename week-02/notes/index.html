Week 02 Notes


  • Note: Much of the topics in this chapter were a review for me. But I wanted to take some notes of things that I find important to remember, even if I understand the concept.
  • Template literals use back ticks (` `) and allow for interpolation of JavaScript code. The JavaScript expression is placed inside curly braces with a dollar sign in front of them - ${ }.
    While I understand this concept. My goal of this note is to remind myself to take advantage of it more often, because I tend to use the concatenation method more often, which I can get a bit unruly when it gets large.
  • It is important to remember when deleting items from an array, if using the delete operator, an index will still remain in the array. It will be given a value of undefined.
    For example:
    delete movies[2];
    ["Jurassic Park", "Back To The Future", undefined]
  • Destructuring as array is the concept of taking values out of an array and presenting them as individual values. We can assign multiple values at the same time.
    For example:
    const [x,y] = [1,2];
    When we check the value of x, we get 1. And likewise, checking the value of y, we get 2.
  • Something I often forget - using a switch/case statement. This is especially useful when needing multiple if and else statements. REMEMBER to use break; after each case statement you provide.
  • Nested for loops have been tricky for me at times.
    Remember, the inner loop will run ALL THE WAY THROUGH before the next step in the outer loop occurs.
  • *Good to remember* - ALL functions return a value. if it is not explicitly stated, the function will return undefined
  • Using Function Expressions is a way of defining a function and assigning it to a variable. This differs from Function Declarations because you are not assigning the function to a variable
  • Callbacks - these are functions that are passed as an argument to another function.
    For example:
    function sing(song, callback) {
    console.log(`I'm singing along to ${song}.`);
    callback();
    }
  • Arrow functions are useful in a few ways from regular functions:
    1. They are a new (as of ES6) shorthand notation to help shorten your code.
    2. They can be used for writing anonymous functions. Anonymous functions are handy for times when the function will only be used once.
  • Function Declarations can be invoked BEFORE they are defined because they are "hoisted" to the top of the scope.
    However, Function Expressions cannot be invoked until AFTER they are defined.
