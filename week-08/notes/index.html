Week 08 Notes


  • Transforms and Transitions
  • The Transform property allows us to manipulate the appearance of any element on the page. We can use:
    - translate
    - rotate
    - scale
    - skew
  • Transforms do not work on inline elements. However you can make that work by adding display: inline-block; to the element.
  • Translate allows us to move elements up, down, left, or right, in the format - translate(x, y).
    An example of translate looks like this:
    transform: translate(45px, -45px)
    This would move the element right and up by 45 pixels each.
  • Scale allows us to grow and shrink an element, in the format - scale(x, y).
    An example of scale looks like this:
    transform: scale(.5, 2);
    This would shrink the width of the element by half, and this would double the height of the element.
  • Rotation allows us to rotate an element by a specified angle, in the format - rotate(25deg);
    An example of rotate looks like this:
    transform: rotate(25deg);
    This would rotate our element by 25 degrees.
  • Skew allows us to skew an element along its x and y axis, in the format - skew(30deg, 90deg)
  • Transition contains 4 properties, they are:
    - transition-property
    - transition-duration
    - transition-timing-function
    - transition-delay
  • We can use shorthand declaration to target each of those properties in a single statement, like this:
    div {
    transition: transform 2s ease-out 50ms;
    }
    Examples of those transitions can be found here: W3Schools - Transitions
  • Animations will animate elements over time. They have much more functionality over typical transitions because unlike transitions only being able to define starting and ending states, animations allow you to control each step of an animation via keyframes.
    To create an animation, you need to use the @keyframes rule. The beginning of creating an animation would look something like this:
    @keyframes myAnimation {
    /* put animation keyframes here */
    }

    There is a lot that goes along with animations. For examples of animations, they can be found here: W3Schools - Animations
  • It's important to note that just because you can make CSS do all types of fun and crazy animations, we need to hold ourselves back in the professional world. We should use transitions and animations where it makes sense, but to really restrain ourselves from going overboard and make our professional website look/feel tacky.


  • Canvas, SVG, and Drag and Drop
  • Canvas was first developed by Apple. It allows you to draw anything that you can imagine, all through JavaScript. Canvas allows you to draw shapes, lines, arcs, text, gradients, patterns. You can also manipulate pixels in images and video as well.
  • Canvas uses the coordinate system, which works like this:
    The top-left corner of the canvas has the coordinates (0, 0). If the canvas is 200px by 200px, then the bottom-right corner of the canvas would contain the coordinates (200, 200).
  • SVG stands for Scalable Vector Graphic. It is a specific file format which will allow you to describe vector graphics using XML.
  • An important feature of vector images is that they will preserve their quality even as you enlarge or shrink the image.
  • A great open source tool that you can use to create SVG images is called Inkscape, and you can visit the site HERE to download the program
  • What's the difference between using Canvas or SVG, since both of these technologies allow you to draw custom shapes, paths, and fonts?

    - Canvas allows for pixel manipulation, however, canvas operates in "immediate mode". This means that you are unable to simply add more to the canvas you're currently working on. Each time a shape is finished drawing, canvas no longer has access to that shape. It doesn't persist as an object that you can modify. So if you want to add to what you've already drawn on the canvas, you need to redraw the new shape from scratch.

    - Canvas is much faster than SVG, due to the fact that the DOM does not have access to what is drawn on the canvas. This would make canvas a better option if you want to design a game that requires a lot of animations.

    - SVG's mode is called "retained mode". This means that what you draw to SVG, it is accessible via the DOM. The structure of the image is preserved in the XML document.

    - Because SVG is a file format - instead of a set of methods which allow you to dynamically draw on a surface - you are unable to manipulate SVG images the way you can manipulate pixels on canvas. For example, it would be impossible to use SVG to convert a color video to black and white, which is possible with canvas.

    - A good rule of thumb: If you want to paint pixels to the screen, without needing to retrieve and modify your shapes, use Canvas. If you need to access and change specific aspects of your graphics, SVG will most likely be better.

  • It is important to note: This API is unsupported by Android and iOS.
