Week 05 Notes


  • Testing and Debugging
  • Errors: Caused when something goes wrong in the program, they are usually caused by one of the following:
    - System error - A problem with the system or external devices with which the program is interacting
    - Programmer error - The program contains incorrect syntax or faulty logic; something like a typo
    - User error - the user has entered data incorrectly, which the program is unable to handle
  • Exceptions: An error that produces a return value that can then be used by the program to deal with the error.
    - An example would be trying to call a method that does not exist. This will give you a reference error that raises an exception
  • Warnings: Can occur if there's an error in the code that isn't enough to cause the program to crash. This will allow the program to run after a warning occurs
    - An example would be when assigning a value to a variable that hasn't been declared
  • Strict Mode will produce more exceptions and warnings, as well as prohibit the use of some deprecated features. Strict mode will consider poorly written code as actual errors. Strict mode is used to encourage better quality of JavaScript.
    To enable strict mode, you simply add the line of code "use strict"; to the first line of the JavaScript file
  • To apply a JavaScript module to your HTML page, you would write this code into the header of your HTML page:
    >script type="module" src="main.js">>/script>
  • You can create a module object, which can let you import all of a JavaScript module features from a specific module with the following code:
    import * as Module from "./modules/module.js";
    By using the "*", this will grab all of the exports available inside of the module.js file. This would allow you to then call any of the available functions, for example:
    Module.function1();
  • When using a try/catch block, you can also use a finally block. The finally block will always be executed after the try or catch block, regardless of whether an exception occurred or not. This is useful if you want to some code to run in either case (try or catch)
  • Test-driven Development (TDD) - This is the process of writing tests before any actual code. First, you write tests that will fail (since there is no code to test at first), then you write some code that will allow the tests to pass, the lastly, the code is refactored to make it faster, more readable, and remove any repetition.
  • Jest is a TDD framework created by Facebook. It makes it easy to create and run tests.
    You start by creating a js file that ends in test.js. For example: mycode.test.js
    Next, you write a test() function inside the file, and pass the test function parameters depending on what code you want tested.
    When you run jest, all files that end with "test.js" within the folder you are testing will be tested and provide you the results
