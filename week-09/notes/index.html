Week 09 Notes


  • The Window Object
  • In a browser environment, the global object is the window object. This represents the browser window that contains a web page.
  • window.location is a property of the window object. This property is an object that contains information about the URL of the current page. Using the href property (window.location.href) will return the full URL as a string.

    hostname property (window.location.hostname) will return a string of the name of the domain of the current URL.

    pathname property (window.location.pathname) will return a string of the path that follows the domain name.

    search property (window.location.search) will return a string that begins with a "?" and then followed by the query string parameters of a URL. It will return an empty string if there are no query parameters.
  • window.history is a property that can be used to access information about any previously visited pages in the current browser session. With this history object, it has a number of properties/methods that go along with it.
  • You can use the window method window.open() to open a new window. This method takes the URL you wish to direct to as the first parameter, and the window title as the second parameter.
  • window.screen object contains information about the screen the browser is displayed on, such as height/width of screen, or even color depth.
  • window.document object contains properties and methods that deal with the page that has been loaded into the window.
  • Cookies are small files that are saved locally on a user's computer. A restriction of cookies is that they can only be read by a web page from the same domain that set them. Cookies are also limited to storing up to 4kb of data and only 20 cookies are allowed per domain.
  • To create a cookie, you use the "cookie" property on the document object and give it a key/value pair, like this:
    document.cookie = "name=Billy Bob";
  • By default, cookies are "session cookies". This means they are deleted once the browser session is finished (the user closes the window). You can set an expiration date on the cookie by using a key/value pair with "expires" as the key name and a date as the value, like this:
    const expiryDate = new Date();
    const tomorrow = expiryDate.getTime() + 1000 * 60 * 60 * 24;
    expiryDate.setTime(tomorrow);
    document.cookie = `name=Batman; expires=${ expiryDate.toUTCString()}`;
  • You can secure cookies by adding ; secure to the end of a string for a cookie. This will ensure it's only transmitted over a secure HTTPS network.
  • In order to delete/remove a cookie, you need to set it to expire at some time in the past.
  • window.setTimeout() method can be used to invoke a function after a set amount of time. It can also be canceled by using the clearTimeout() method.


  • HTML5 APIs
  • Using the data- attribute is a way of embedding data in a web page using custom attributes that are ignored by the browser. These attributes are also private to a page, so they are not intended to be used by an external service.
  • To use the data- attribute, you must follow the following format:
    - Start with data-
    - Contain only lowercase letters, numbers, hyphens, dots, colons, or underscores.
    - Include an optional string value.
    In your HTML document, you would apply the attribute like this:
    div id="hero" data-powers="flight super-speed">Superman /div
  • You could access the "data-" attribute example above like this:
    const superman = document.getElementById("hero");
    const powers = superman.dataset.powers;
    console.log(powers);
  • The HTML5 specification contains a number of built-in APIs that help gain access to different things such as cameras, batteries, geolocation, etc.
  • HTML5 Web Storage API can come in the form of localStorage or sessionStorage. localStorage is persistent and will remain on the local computer browser so long as the storage is not cleared, whereas sessionStorage will only persist so long as the session remains open and the browser is not closed.
  • The HTML5 Web Storage API is similar to cookies, however it has fewer restrictions than cookies, has more storage capacity, and is generally easier to use than cookies. This storage is perfect for storing information about users, as well as application-specific information that can be used later during future sessions.
  • Web Storage API has some major differences to cookies, they are:
    - Information stored is NOT shared with the server on every request.
    - Information is available in multiple windows of the browser (so long as the domain is the same)
    - Storage capacity for Web Storage APIs does not have a set limit, unlike 4KB for cookies.
    - Any data stored does not automatically expire as it does with cookies. For this reason, cookies might be a better choice for something like showing a popup once a day.
  • Geolocation API is another built-in HTML5 API which allows for obtaining the geographical position of the device.
  • To access Geolocation API, you will use the navigator object. An example of using the "navigator" object would look something like this:
    navigator.geolocation.getCurrentPosition();
  • Web Workers allow processes to be run in the background. This allows for adding support for concurrency in JavaScript.
    In order to create a Web Worker, you would create it like this:
    const worker = new Worker("task.js");
  • The Service Worker API allows a worker script to run in the background with the added benefit of being able to intercept network requests.
  • Websockets are a new protocol for communicating over the internet, and allow for real-time, two-way communication.
  • The Notification API allows you to display notifications on the user's system.
  • The audio and video elements can be used to embed audio tracks and video clips in a web page. There is also a Media API that can be used to control the playback using JavaScript.
